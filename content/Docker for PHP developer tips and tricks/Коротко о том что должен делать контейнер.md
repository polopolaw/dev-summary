#### Контейнер после старта должен стать healthy без доп манипуляций с контейнером

Другими словами мы не хотим чтобы приходилось использовать exec для манипуляций, перезапустить сервер, дозаписать кэш и т.д. Он должен быть готов к запуску.

В некоторых Symfony репозиториях  прописывают в Entrypoint:

● composer install


● database migration


● configure


● warmup cache

Но чем меньше таких инструкций тем лучше. Старайтесь выносить их в docker build чтобы контейнер стартовал максимально быстро.

> [!Warning] Внимание
> 
> Автор на данном этапе еще не уверен во всех этих рекомендациях.
